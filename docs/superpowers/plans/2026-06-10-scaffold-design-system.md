# AlgoClar Scaffold + Design System — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap AlgoClar Next.js project: tooling, design tokens, base component library, header/footer, route stubs, README.

**Architecture:** Next.js 15 App Router, server components by default. Design tokens live exclusively in `app/globals.css` (`:root` + `.dark` HSL channels). Tailwind v4 `@theme inline` maps those variables to utility classes. shadcn/ui provides primitive components. All custom components consume tokens via Tailwind utilities — zero hardcoded colors.

**Tech Stack:** Next.js 15, TypeScript strict, pnpm, Tailwind CSS v4, shadcn/ui, next-themes, next/font (Sora + IBM Plex Sans + JetBrains Mono), lucide-react, ESLint + Prettier.
