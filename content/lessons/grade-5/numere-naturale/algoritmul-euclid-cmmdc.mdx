<LessonHook>
Vrei să împarți o tablă de 84 cm și una de 36 cm în bucăți egale, cât mai mari, fără risipă. Răspunsul este cel mai mare număr care divide ambele lungimi — CMMDC. Algoritmul lui Euclid îl calculează în câteva scăderi succesive (sau împărțiri) — și e folosit de mai bine de 2300 de ani.
</LessonHook>

## Ce este CMMDC?

**Cel Mai Mare Divizor Comun** al lui `a` și `b` este cel mai mare număr care divide exact ambele.

**Exemplu:** CMMDC(84, 36) = 12, deoarece 84 = 12×7 și 36 = 12×3.

## Algoritmul lui Euclid

**Proprietate cheie:** `CMMDC(a, b) = CMMDC(b, a % b)`

Aplicăm repetat până când `b` devine 0:

```
CMMDC(84, 36):
  84 % 36 = 12 → CMMDC(36, 12)
  36 % 12 = 0  → CMMDC(12, 0)
  b = 0 → rezultat: 12
```

<ObservationBox>
La fiecare pas, cel mai mic argument se înjumătățește cel puțin o dată. Asta garantează că algoritmul se termină rapid — în O(log(min(a,b))) pași, chiar și pentru numere foarte mari.
</ObservationBox>

## Implementare C++ — iterativă

```cpp
#include <iostream>
using namespace std;

int cmmdc(int a, int b) {
    while (b != 0) {
        int rest = a % b;
        a = b;
        b = rest;
    }
    return a;
}

int main() {
    int a, b;
    cin >> a >> b;
    cout << "CMMDC(" << a << ", " << b << ") = " << cmmdc(a, b) << endl;
    return 0;
}
```

**Exemplu:** Input `84 36` → `CMMDC(84, 36) = 12`

## Varianta recursivă

```cpp
int cmmdc(int a, int b) {
    if (b == 0) return a;       // caz de baza
    return cmmdc(b, a % b);     // pas recursiv
}
```

Elegantă, dar consumă mai multă memorie (stiva de apeluri).

## Complexitate

| Caz | Timp | Spațiu |
|-----|------|--------|
| Iterativ | O(log(min(a,b))) | O(1) |
| Recursiv | O(log(min(a,b))) | O(log n) — stivă |

<MistakeBox>
**Greșeala 1:** Confunzi ordinea la pasul de reducere. `CMMDC(a, b) = CMMDC(b, a % b)` — noul prim argument e `b`, nu `a % b`.

**Greșeala 2:** Aplici algoritmul cu `a = 0`. `CMMDC(0, b) = b` prin definiție. Dacă inputul poate conține 0, verifică separat.

**Greșeala 3:** La varianta iterativă, uiți să salvezi `rest = a % b` înainte de `a = b`. Dacă scrii `a = b; b = a % b`, al doilea `a` e deja suprascris — rezultat greșit.
</MistakeBox>
